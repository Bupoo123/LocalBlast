1. 软件简介

LocalBlast 是一个本地化的BLAST序列比对工具，支持blastn（DNA序列比对）功能。无需连接NCBI服务器，所有比对在本地完成，保护数据隐私。

主要特点：
  • 🧬 支持DNA序列比对（blastn）
  • 🎯 预置94种常见病原体参考序列
  • 📊 生成美观的HTML结果页面
  • 🚀 本地运行，数据安全
  • 📦 支持批量文件处理

2. 系统要求

2.1 必需软件

  • Python 3.7+
  • BLAST+工具 - NCBI BLAST+命令行工具

2.2 安装BLAST+

macOS:

brew install blast

Linux (Ubuntu/Debian):

sudo apt-get update
sudo apt-get install ncbi-blast+

Windows:

从NCBI官网下载安装包：
https://blast.ncbi.nlm.nih.gov/Blast.cgi?CMD=Web&PAGE_TYPE=BlastDocs&DOC_TYPE=Download

3. 快速开始

3.1 启动服务

在项目目录下运行：

python3 blast_app.py

或使用启动脚本：

./start_blast.sh

3.2 访问界面

打开浏览器访问：http://localhost:5001

4. 单序列比对

4.1 输入序列

在查询序列输入框中输入或粘贴DNA序列：

  • 支持FASTA格式（带或不带头部）
  • 支持纯序列文本
  • 序列只能包含A、T、C、G字符
# FASTA格式示例
>Query
ATGCGATCGATCGATCGATCG

# 或直接输入序列
ATGCGATCGATCGATCGATCG

4.2 选择比对模式

模式1：指定物种比对

  • 从下拉菜单中选择要比对的物种
  • 系统只与该物种进行比对
  • 适合已知目标物种的情况

模式2：全物种比对（推荐）

  • 不选择任何物种（保持默认选项）
  • 系统自动与所有94个物种进行比对
  • 自动显示得分最高的最佳匹配结果
  • 适合未知序列的物种鉴定
提示：全物种比对使用统一数据库优化，速度非常快（5-15秒），比逐个比对快20-100倍！

4.3 查看结果

比对完成后，结果页面会显示：

  • 查询序列长度
  • 匹配的物种信息
  • 比对统计：Max Score、Total Score、Query Cover、E value、Per. Ident等
  • 详细的比对结果表格

5. 批量文件处理

5.1 访问批量上传页面

点击主页右上角的"📦 批量上传比对"链接，或直接访问：http://localhost:5001/batch

5.2 上传文件

  • 支持上传多个 .seq 格式文件
  • 可以点击上传区域选择文件
  • 也可以直接拖拽文件到上传区域
  • 文件列表会显示文件名和大小
  • 可以点击 × 移除不需要的文件

5.3 .seq文件格式

.seq文件格式说明：

  • 每行包含序列片段
  • 可能包含行号（会自动过滤）
  • 序列字符：A、T、C、G
  • 系统会自动合并所有行的序列
# .seq文件示例
1 TACYAATTCTTCGCCCGTTTATACGGTCGTAACTATTGATGTCCGCCCGTTCTCGAACGGATGAATATTGGGGATCGCTT
2 GACCATACATCGCCGTTATTTAATGTCATTGAACCACGTATGTTTCTATCAAATATAGAAGGAGTGCGTTCAGAATAAGG
3 CGCATTYA

5.4 批量处理流程

  1. 上传所有 .seq 文件
  2. 点击"开始批量比对"按钮
  3. 系统自动处理每个文件：
    • 解析序列
    • 与所有物种比对（使用统一数据库）
    • 找出最佳匹配
    • 生成HTML结果文件
  4. 处理完成后，点击"下载所有结果图片（ZIP）"按钮
  5. 下载ZIP文件，包含所有结果HTML文件
注意：批量处理可能需要一些时间，取决于文件数量和序列长度。请耐心等待。

6. 结果解读

6.1 比对统计指标

指标 说明
Max Score 最高比对得分，数值越高表示匹配越好
Total Score 总比对得分
Query Cover 查询序列覆盖率，表示查询序列有多少百分比被比对上了
E value 期望值，数值越小表示匹配越显著（通常<0.05认为有意义）
Per. Ident 序列相似度百分比，数值越高表示序列越相似
Acc. Len 参考序列长度

6.2 结果判断

  • 高置信度匹配：Per. Ident > 95%，E value < 1e-50
  • 中等置信度：Per. Ident 80-95%，E value 1e-10 到 1e-50
  • 低置信度：Per. Ident < 80%，需要进一步验证

7. 常见问题

7.1 BLAST未找到

问题：提示"BLAST+未安装"

解决：

  • 检查BLAST+是否正确安装：blastn -version
  • 确保BLAST+在系统PATH中
  • macOS用户：brew install blast

7.2 端口被占用

问题：端口5000被占用（macOS的AirPlay Receiver）

解决：程序默认使用5001端口，无需修改。如需使用5000端口，可在系统设置中关闭"隔空播放接收器"

7.3 序列比对失败

问题：比对失败或没有结果

解决：

  • 检查序列格式是否正确（只包含ATCG）
  • 确保序列长度足够（建议>50bp）
  • 检查序列是否包含特殊字符
  • 查看控制台错误信息

7.4 批量处理速度慢

问题：批量处理很慢

说明：系统已使用统一数据库优化，速度比逐个比对快20-100倍。如果仍感觉慢，可能是：

  • 文件数量很多（建议分批处理）
  • 序列很长（比对时间与序列长度相关）
  • 系统资源不足

8. 技术说明

8.1 比对算法

使用NCBI BLAST+的blastn算法，这是标准的DNA序列比对工具。

8.2 性能优化

  • 统一数据库：所有物种序列合并到一个数据库，只需创建一次
  • 单次比对：全物种比对只需执行一次blastn，而非94次
  • 速度提升：比传统方法快20-100倍

8.3 数据安全

所有比对在本地完成，数据不会上传到任何服务器，完全保护隐私。
